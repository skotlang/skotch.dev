Skotch implements a growing subset of the Kotlin language. This page
summarizes what works today and what is coming next.

<Aside type="note">
Estimated coverage: roughly 45% of the Kotlin language specification.
220+ test fixtures are validated across JVM, DEX, LLVM IR, klib, and
native targets.
</Aside>

## Supported features

### Functions

Top-level functions with parameters, return types, expression bodies,
extension receivers, local (nested) functions, and mutual recursion.

```kotlin
fun greet(name: String): String = "Hello, $name!"

fun Int.isEven(): Boolean = this % 2 == 0

fun main() {
    println(greet("world"))
    println(42.isEven())
}
```

### Variables

Immutable `val` and mutable `var` with type inference from literals or
explicit type annotations.

```kotlin
val message = "hello"
var counter = 0
val typed: Int = 42
```

### Types

| Type | Supported | Notes |
|------|-----------|-------|
| `Int` | Yes | 32-bit, all literal forms (decimal, hex, binary, `L`, underscores) |
| `String` | Yes | Regular, raw (`"""`), templates (`$x`, `${expr}`) |
| `Boolean` | Yes | `true`, `false` |
| `Char` | Yes | `'A'`, escape sequences (`'\n'`, `'\t'`, `'\\'`) |
| `Unit` | Yes | Implicit return type for void functions |
| `Any` | Yes | Top type |
| `Double` | Yes | `3.14`, `2.5e10`, `1.0f` — full arithmetic, `println(Double)` |
| `null` | Yes | `null` literal, `println(null)` |
| `Long` | Yes | `100L`, `0xFFL` — full 64-bit arithmetic, negation, values beyond Int range |
| `T?` (nullable) | Partial | `?:` elvis works; `?.` and `!!` parsed but not yet compiled |

### Operators

Arithmetic (`+`, `-`, `*`, `/`, `%`) on `Int`, `Long`, and `Double`,
comparison (`==`, `!=`, `<`, `>`, `<=`, `>=`), logical (`&&`, `||`),
unary (`-`, `!`), and compound assignment (`+=`, `-=`, `*=`, `/=`, `%=`).

String concatenation works with `+` between strings and between string
and int.

### Nullable types and elvis operator

Functions can accept nullable parameters (`String?`). The elvis operator
(`?:`) provides a default value when the left-hand side is null:

```kotlin
fun greet(name: String?): String = name ?: "anonymous"

fun first(a: String?, b: String?): String = a ?: b ?: "none"

fun main() {
    println(greet("Alice"))    // Alice
    println(greet(null))       // anonymous
    println(first(null, null)) // none
}
```

### Control flow

All core control-flow constructs are implemented:

- **if/else** as both statement and expression
- **else-if chains** for multi-branch conditions
- **when** with subject, without subject, comma patterns, `in range`
  patterns, string/int matching, `else` default, and nesting
- **for** loops over `Int` ranges (`for (i in 0..9)`)
- **while** and **do-while** loops
- **break**, **continue**, and **return** (including early return and
  guard clauses: `if (n <= 1) return n`)
- **try/finally** — finally block always executes after try body

```kotlin
fun classify(n: Int): String = when {
    n < 0 -> "negative"
    n == 0 -> "zero"
    n in 1..10 -> "small"
    else -> "large"
}
```

### String templates

Both `$identifier` and `${expression}` interpolation forms work in regular
and raw strings.

```kotlin
val x = 42
println("The answer is $x")
println("Double: ${x * 2}")
```

### String methods

String instance methods are compiled to JVM `invokevirtual` calls:

```kotlin
val s = "Hello, World!"
println(s.length)                  // 13
println(s.uppercase())             // HELLO, WORLD!
println(s.lowercase())             // hello, world!
println(s.isEmpty())               // false
println(s.trim())                  // Hello, World!
println(s.substring(7))            // World!
println(s.substring(0, 5))         // Hello
println(s.contains("World"))       // true
println(s.startsWith("Hello"))     // true
println(s.endsWith("!"))           // true
println(s.indexOf("World"))        // 7
println(s.replace("World", "Kotlin")) // Hello, Kotlin!
println(42.toString())             // 42
```

### Init blocks

Class `init` blocks execute during construction:

```kotlin
class Logger(val tag: String) {
    init {
        println("Logger initialized: $tag")
    }
    fun log(msg: String) { println("[$tag] $msg") }
}
```

### Built-in functions

`println()` is a compiler intrinsic with overloads for `Int`, `Double`,
`String`, `Boolean`, `null`, and no-argument (prints a blank line).

### Const val

Compile-time constants are inlined at their call site:

```kotlin
const val PI = 3.14159
const val APP_NAME = "skotch"

fun main() {
    println(PI)        // 3.14159
    println(APP_NAME)  // skotch
}
```

### Visibility modifiers

`private`, `internal`, `protected`, `open`, and `abstract` are
recognized by the parser. They don't yet enforce access control but
allow real Kotlin code to be parsed without errors.

### Comments

Line comments (`//`) and block comments (`/* */`) are supported.

## Not yet supported

<Aside type="caution">
The following features are parsed by the front-end but produce a
compiler diagnostic if you try to use them. They are planned for
future releases.
</Aside>

| Feature | Status |
|---------|--------|
| Safe call operator (`?.`) | Parsed; needs field/method dispatch on nullable receiver |
| Non-null assertion (`!!`) | Parsed; passthrough (no NullPointerException yet) |
| Type checks (`is`/`!is`) | Parsed; stub lowering (always true) |
| Type casts (`as`/`as?`) | Parsed; passthrough lowering |
| Try/catch | Parsed; catch needs JVM exception tables (try/finally works) |
| Throw expression | Parsed; needs `athrow` opcode |
| Data classes | Needs synthesized `equals`/`hashCode`/`toString`/`copy` |
| Objects and interfaces | Needs INSTANCE field synthesis and `invokeinterface` |
| Sealed classes | Needs exhaustiveness checking |
| Enums | Needs enum iteration and synthetic methods |
| Generics | Needs type parameter bounds, variance, type erasure |
| Lambdas and closures | Needs closure capture, `it` parameter, SAM conversion |
| Collections (`listOf`, `map`, `filter`) | Blocked on generics and lambdas |
| Default and named arguments | Needs synthetic `$default` overloads |
| Varargs | Needs `vararg` parameter and spread `*` |
| Destructuring declarations | Needs `componentN()` synthesis |
| Coroutines | Needs CPS state-machine transform |
| Annotations | Needs retention and reflection metadata |
| Operator overloading | Needs `plus`, `minus`, `compareTo`, `invoke` dispatch |
| Type aliases | Easy — erased at compile time |

## What you can build today

Skotch handles procedural and object-oriented Kotlin well: console
programs, algorithms, math utilities, classes with constructors and
methods, and null-safe code using the elvis operator. Programs that
use functions, control flow, basic types, Double arithmetic, and
nullable parameters compile and run correctly on all five targets.

### Example: Fibonacci sequence

```kotlin
fun fibonacci(n: Int): Int {
    if (n <= 1) return n
    var a = 0
    var b = 1
    for (i in 2..n) {
        val temp = a + b
        a = b
        b = temp
    }
    return b
}

fun main() {
    for (i in 0..10) {
        println(fibonacci(i))
    }
}
```

### Example: Recursive GCD

```kotlin
fun gcd(a: Int, b: Int): Int {
    if (b == 0) return a
    return gcd(b, a % b)
}
```

### Example: Null-safe parameter handling

```kotlin
fun greet(name: String?): String = name ?: "World"
fun first(a: String?, b: String?): String = a ?: b ?: "none"
```

Programs that depend on the Kotlin standard library collection APIs,
generics, lambdas, or third-party libraries will need to wait for
those features to land.
